## 📅 Fases de Implementação com Checkboxes
### Fase 1: Estrutura, Cabeçalho e Identidade (Alicerce)
- \[ ] \*\*Identidade Visual e Assets\*\*    - \*\*O que fazer:\*\* Preparar logomarca em alta resolução (.svg ou .png transparente).    - \*\*Por que:\*\* Garantir que o site tenha uma aparência profissional em qualquer tela.- \[ ] \*\*Definição das 5 Categorias Principais\*\*    - \*\*O que fazer:\*\* Listar os nomes exatos das categorias para o Mega Menu.    - \*\*Ferramenta:\*\* Planejamento estratégico de conteúdo.- \[ ] \*\*Configuração do Ambiente Next.js 15 + Tailwind 4\*\*    - \*\*O que fazer:\*\* Criar o repositório no GitHub e configurar a estrutura de pastas profissional.    - \*\*Dica:\*\* Use o comando \`npx create-next-app@latest\` para iniciar.- \[ ] \*\*Desenvolvimento do Layout Base (Main Layout)\*\*    - \*\*O que fazer:\*\* Criar a "moldura" do site (Header e Footer) que se repete em todas as páginas.- \[ ] \*\*Engenharia do Mega Menu Inteligente\*\*    - \*\*O que fazer:\*\* Codificar o menu que lê os arquivos Markdown e organiza as categorias automaticamente.- \[ ] \*\*Implementação do Design "Glassmorphism" no Cabeçalho\*\*    - \*\*O que fazer:\*\* Aplicar transparência, ícones neon e a logo usando Tailwind 4.- \[ ] \*\*Integração Inicial com Firebase Auth (SSO)\*\*    - \*\*O que fazer:\*\* Configurar a conexão com o Firebase para permitir o Login Único.    - \*\*Pilar A:\*\* Iniciar o sistema de "Comunidade Blindada" restringindo o acesso inicial.- \[ ] \*\*Sistema de Otimização Automática de Imagens\*\*    - \*\*O que fazer:\*\* Configurar o componente \`Next/Image\` para converter imagens para .webp ou .avif automaticamente.- \[ ] \*\*\[NOVO] 🚀 Engenharia do Console de Criação e Manutenção (Decap CMS)\*\*    - \*\*Objetivo:\*\* Criar um painel administrativo visual (\`/admin\`) para escrever e publicar artigos, eliminando a necessidade de editar arquivos Markdown manualmente no código.    - \*\*Tecnologia:\*\* Decap CMS (Git-based).    - \*\*Subtarefas:\*\*        - \[ ] \*\*1. Configuração de Autenticação via GitHub:\*\*            - \*\*O que fazer:\*\* No painel da Vercel (ou diretamente no GitHub), criar um "OAuth App".            - \*\*Por que:\*\* Para permitir que o Decap CMS se autentique com sua conta do GitHub e tenha permissão para salvar (fazer \`commit\`) os artigos no seu repositório.        - \[ ] \*\*2. Estruturação dos Arquivos do Console:\*\*            - \*\*O que fazer:\*\* Criar a pasta \`/public/admin/\` e, dentro dela, os arquivos \`index.html\` (para carregar o script do Decap) e \`config.yml\` (o cérebro do console).            - \*\*Por que:\*\* Esta é a estrutura padrão que o Decap CMS usa para "viver" dentro do seu projeto Next.js.        - \[ ] \*\*3. Mapeamento dos Campos do Artigo no \`config.yml\`:\*\*            - \*\*O que fazer:\*\* Traduzir cada campo do seu \`front matter\` (título, autor, data, imagem, tags, etc.) para a sintaxe do \`config.yml\`, definindo os \`widgets\` corretos para cada um (texto, data, imagem, lista, markdown).            - \*\*Por que:\*\* É isso que transforma a estrutura do seu \`front matter\` em um formulário visual e intuitivo.        - \[ ] \*\*4. Configuração da Galeria de Mídia:\*\*            - \*\*O que fazer:\*\* No \`config.yml\`, definir a \`media_folder\` para apontar para a pasta onde as imagens dos artigos serão salvas (ex: \`/public/assets/images/uploads\`).            - \*\*Por que:\*\* Para que, ao fazer upload de uma imagem pelo console, ela seja salva no lugar certo e o caminho seja inserido corretamente no \`front matter\`.        - \[ ] \*\*5. Teste e Validação do Fluxo Completo:\*\*            - \*\*O que fazer:\*\* Acessar a rota \`/admin\`, fazer login, criar um artigo de teste preenchendo todos os campos, salvar e verificar se:                - a) Um novo \`commit\` foi criado no GitHub.                - b) A Vercel iniciou um novo \`build\` automaticamente.                - c) O artigo de teste apareceu no blog publicado.            - \*\*Por que:\*\* Para garantir que todo o fluxo, desde a criação até a publicação, está funcionando perfeitamente.
